# 1Take Camera v0.1 Implementation Plan

> **For agentic workers:** Use superpowers:subagent-driven-development. Steps use checkbox syntax.

**Goal:** Ship a minimal but complete TestFlight-ready camera app with viewfinder, 4 compressor character presets, level metering, clip warning, recording list, and lifecycle handling. Compressor-only DSP (option B); other effect stages deferred to v0.2.

**Architecture:** Same SPM-package-driven layout as v0.0. All v0.1 features added inside `CameraFeature/`. Compressor character is selected at recording-start time and embedded in MP4 metadata. UI is portrait-only, full-screen viewfinder + bottom strip.

**Tech Stack:** SwiftUI, AVFoundation, OneTakeDSPCore (CompressorEngine), Photos framework, os.Logger.

---

## Known Issues from v0.0 PoC

1. **A/V drift ≈60ms** (audio shorter than video). Observed in `1TakeCam-20260407-172536.mp4`: video 30.67s, audio 30.61s. Must be diagnosed and fixed before v0.1 ships.
2. **Simulator crash potential** — `AVCaptureDevice.default(.builtInWideAngleCamera, ...)` returns nil on simulator. Currently no guard.
3. **Black screen** — viewfinder not displayed.

## Task 1: Diagnose A/V drift

**Files:** investigation only — `OneTakeCameraPackage/Sources/CameraFeature/Capture/CameraSession.swift`, `MovieWriter.swift`, `Audio/SampleBufferConverter.swift`

- [ ] **Step 1: Add detailed PTS logging**

In `CameraSession.audioOutput(_:didOutput:from:)`, log first/last/total sample count and PTS of every Nth audio CMSampleBuffer (e.g. every 50th). Also log video PTS at the same cadence.

```swift
private var audioBufferCount = 0
// inside audio delegate:
audioBufferCount += 1
if audioBufferCount % 50 == 0 {
    let pts = CMSampleBufferGetPresentationTimeStamp(sampleBuffer)
    let n = CMSampleBufferGetNumSamples(sampleBuffer)
    logger.info("audio[\(self.audioBufferCount)] pts=\(pts.seconds, privacy: .public) samples=\(n, privacy: .public)")
}
```

- [ ] **Step 2: Verify SampleBufferConverter sample-count round-trip**

In `SampleBufferConverter.toSampleBuffer(_:originalTiming:)`, assert that the output `CMSampleBuffer`'s sample count equals the input frame count. If `AVAudioConverter` is doing format conversion that changes sample count (e.g. capture is 44.1k mono → internal 48k stereo), the converted frame count will differ from the original. The converter must preserve the original timing **stamp** but the **sample count** in the output buffer is the post-conversion count, and PTS arithmetic must use the post-conversion sample rate (48k).

- [ ] **Step 3: Check MovieWriter finalize ordering**

`MovieWriter.finalize()` should call `markAsFinished()` on **all** inputs (video + audio) before `finishWriting`. If audio input is marked finished too early (e.g., the last few audio buffers from a still-active capture queue arrive after `markAsFinished`), they're dropped silently → audio stream is shorter than video. Fix: drain the capture queue once, append a final sentinel, then mark finished.

- [ ] **Step 4: Implement the fix**

Most likely root cause is Step 3. Implementation:
1. When user taps stop or 30s timer fires, `CameraSession` captures the current PTS as `stopPTS`.
2. Stop accepting new buffers older than `stopPTS` for both audio and video.
3. Drain in-flight buffers from the capture queue (`captureQueue.sync { }`).
4. Call `MovieWriter.finalize()` which calls `markAsFinished()` on both inputs and then `finishWriting`.

- [ ] **Step 5: Verify fix on device**

Re-record 30s on wichish, copy via `devicectl`, run `ffprobe`. Audio duration must be within ±10ms of video duration.

- [ ] **Step 6: Commit**

```
git add -A
git commit -m "fix(camera): A/V drift — drain capture queue before MovieWriter.finalize"
```

        case .live: "VCA"

- [ ] **Step 2: Make `AudioProcessor` accept the preset at construction and reset state when changed**

```swift
public final class AudioProcessor: @unchecked Sendable {
    private var preset: CompressorPreset
    private var state = CompressorState()

    public init(preset: CompressorPreset) { self.preset = preset }

    public func setPreset(_ new: CompressorPreset) {
        preset = new
        state = CompressorState() // reset envelope
    }

    public func process(left: UnsafeMutablePointer<Float>, right: UnsafeMutablePointer<Float>?, frameCount: Int) {
        CompressorEngine.process(
            left: left, right: right, frameCount: frameCount,
            settings: preset.settings, model: preset.model, state: &state
        )
    }
}
```

## Task 4: Real-time level meter (post-DSP)

**Files:**

- [ ] **Step 1: Add post-DSP level extraction**

After `AudioProcessor.process(...)`, compute peak (max abs) of L+R and store in an atomic-friendly value (`OSAllocatedUnfairLock<Float>` from iOS 16+).

- [ ] **Step 2: `LevelMonitor` — `@Observable` class polling at ~30Hz**

```swift
import Observation
@MainActor
@Observable
public final class LevelMonitor {
    public var peakDB: Float = -60
    private var task: Task<Void, Never>?

    public func start(reading: @escaping () -> Float) {
        task = Task { [weak self] in
            while !Task.isCancelled {
                let v = reading()
                let db = v > 1e-5 ? 20 * log10(v) : -60
                await MainActor.run { self?.peakDB = db }
                try? await Task.sleep(for: .milliseconds(33))
            }
        }
    }

    public func stop() { task?.cancel(); task = nil }
}
```

- [ ] **Step 3: `LevelMeterView` — horizontal bar -60..0 dB**

Use `Rectangle().frame(width:)` mapped from peakDB. Color: green ≤ -12dB, yellow ≤ -3dB, red > -3dB.
